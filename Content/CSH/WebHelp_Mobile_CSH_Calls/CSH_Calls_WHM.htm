CSH Calls for WebHelp Mobile

You can work with your developer (or you can function as the developer yourself) to open specific parts of your documentation from web links. In order to accomplish this, you need to have created context-sensitive Help (CSH) in your Flare project and share the header file information with the developer.

What You Need to Do

what you need to do
  1. Work with your developer to determine how you want to connect the mobile output to the web links.
  2. Create and build your target in Flare.
  3. Provide the developer with the output files, as well as the CSH header file.
  4. Provide the developer with the information in the following topic: CSH Calls for WebHelp Mobile—Developers.
