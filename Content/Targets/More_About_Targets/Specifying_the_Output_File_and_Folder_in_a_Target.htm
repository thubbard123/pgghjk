Specifying the Output File and Folder in a Target

When you build the output for a target in your project, sends the output files to the Output folder where your project is located. It also uses a default name for the main entry file to the output.

However, you can specify a different location and main entry file name for a target's output files.

example

example

Let's say you have a project called "FictionSoftPro" and have stored it in your root C:\ directory. If you create a project based on the "Empty" template and then build a target named "MyDotNetHelp," places the output files in C:\FictionSoftPro\Output\MyName\MyDotNetHelp, with the default main entry file named "Manual.mchelp." If you build the output for an target, places the output files in C:\FictionSoftPro\Output\MyName\MyHtmlHelp, with the main entry file (actually, the only file) named "Manual.chm."

 how to specify the output file and folder in a target
  1. Open the target.
  2. On the General tab of the Target Editor, do one or both of the following.

    • To specify the main entry file name, type the desired name in the Output File field.

      For Word output, the default file type is XML. However, you can enter one of the following extensions at the end of the file name to use that file type: .doc, .docx, .pdf, .xps.

      For FrameMaker output, the default type is BOOK and/or FM. However, you can enter .pdf in the field to use that file type.

      This field affects the name of the .mcbook file (used for viewing XHTML output in the MadCap Book Viewer). It also affects the name of the .book file for FrameMaker output. If you are creating multiple output files using chapter breaks, you can specify the name of each chapter file in the Printed Output tab of the TOC Properties dialog.

      The Output File field is not available for targets.

    • To specify the location of the output files, click in the Output Folder field. Then navigate to and select the location in your computer where you want to place the files.
