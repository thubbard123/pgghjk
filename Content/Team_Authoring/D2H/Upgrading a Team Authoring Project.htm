Upgrading a Team Authoring Project

All users of a team project must have exactly the same version (including build number) of Doc-To-Help installed on their machines. To upgrade the project to a newer version of Doc-To-Help, the administrator must first upgrade the team project in the repository and then each team member can install the newest version of Doc-To-Help.

How to Upgrade a Team Authoring Project

The following steps should be performed by the administrator.

  1. Install the new version of Doc-To-Help.
  2. Double-click C1D2HTeamAdmin.exe. See Team Authoring Administrative Functions.
  3. Select File>Open Team Project to connect to the repository. The Load Remote Project dialog opens.
  4. Select the Repository Type. (If necessary, enter your login credentials.)
  5. Select a team project (or from the drop-down choose Select Team Project to browse for a project) and click OK.
  6. Select Tools>Upgrade Team Project to the Current Doc-To-Help Version.

Once the project is upgraded to the current version of Doc-To-Help, all team members should install the same version of Doc-To-Help, open their projects, and choose File>Team Authoring>Get Latest Version of the Project.

 Note

For other tasks, see Team Authoring Administrative Functions.
