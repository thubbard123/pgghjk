Adding Widgets to a Dashboard

How to Add a Widget to a Dashboard

    • Home Page Click . The Home page opens.

      OR

    • Project Dashboard Click . The Projects page opens.

      In the Name column, select a project. The Project dashboard opens.

  2. In the local toolbar click . The Select Widgets to Add dialog opens.

  3. Select the widget(s) you want to add to the dashboard.

    Available Widgets

    Expand any of the following sections to learn about the widgets you can add to the dashboard.

    Build History

    The Build History widget displays a graph of the recent build history for the selected target. Select the project and target you want to view from the widget menu . Additionally, 10 builds are displayed by default; you can also change this setting from the widget menu .

    By default, the most recent build is selected in the widget, and information about that build is shown below the graph. Click any bar in the graph to display information for a different build.

    In this widget, you can view the following:

    • Build Progress Each bar in the graph indicates the amount of time it took to build the target (i.e., taller bars took more time to build), as well as the target's status. The color of the bar is different depending on what happens with the build. If the build finishes with no errors or warnings (although there may be ignored warnings), the bar turns dark green. If the build finishes with no errors (although there may be warnings), the bar turns yellow. If an error prevents the build from finishing, the bar is red. If you manually stop the build, the bar is gray.
    • Build Status The status of the build (e.g., completed, canceled).
    • By The person who queued the build.
    • Duration The length of time it took to complete the build.
    • Warnings / Ignored Warnings / Errors The number of issues found while building the project. For more information, click Build Log to open the build log. See Opening the Build Log.
    • Queued The date and time that the build was queued.
    • Started The date and time that the build started to build.
    • Finished The date and time that the build finished building.

    To open the build log for the selected build, click Build Log. To view the selected build, click View. To download the selected build, click Download.

    See Opening the Build Log, Viewing Output, and Downloading Builds.

    Data will not display in this widget until you select a project and target, and until you generate the target.

         Builds Activity

    The Builds Activity widget shows all of the builds that are in a state of failed, completed, canceled, or queued. The most recent build appears at the top of the widget.

    By default, this widget shows builds for all projects, but you can use the widget menu to filter the widget to show the builds for a single project (see Filtering Widgets). Additionally, 10 builds are displayed by default; you can also change this setting from the widget menu .

    In this widget, you can view the following:

    • Start Time The date and time the build was started.
    • Target The output icon (e.g., PDF, HTML5) and name of the target. Click the name of the target to open the Build Management page. See About Builds.
    • Project The name of the project. Click the name to open the Project dashboard.
    • Status The current build status (e.g., finished, running, terminated).
    • State The state of the build (e.g., building, canceled, build complete).
    • Build Progress The progress of the build towards completion. This column displays a colored bar, depending on what happens with the build. When a build is in progress, the bar is light green. If the build finishes with no errors or warnings (although there may be ignored warnings), the bar turns dark green. If the build finishes with no errors (although there may be warnings), the bar turns yellow. If an error prevents the build from finishing, the bar is red. If you manually stop the build, the bar is gray.
    • Warnings / Ignored Warnings / Errors The number of issues found while building the project. For more information, click the project name to open the Project dashboard, then open the build log. See Opening the Build Log.

    Data will not display in this widget until you generate a target.

         Live Builds

    The Live Builds widget shows you all of the builds that are currently set as live. By default, this widget shows builds for all projects, but you can use the widget menu to filter the widget to show the builds for a single project (see Filtering Widgets).

    In this widget, you can view or do the following:

    • Target Name The output icon (e.g., PDF, HTML5) and name of the target. Click the name of the target to open the Build Management page. See About Builds.
    • URL The vanity URL for the live build. Click the URL to open the live build in a new tab. See Setting Builds to Live or Keep.
    • Download Downloads the output. Multi-file outputs are downloaded as a ZIP file. See Downloading Builds.
    • Live Date The date the build was set as live.

    Data will not display in this widget until you set a target to "Live."

         Project Bookmarks

    The Project Bookmarks widget allows you to quickly jump to the Project dashboard for your favorite projects. To add projects to Project Bookmarks widget, select them from the widget menu . Click any project name in the widget to open the Project dashboard for that widget.

    Data will not display in this widget until you select a project.

         Project Properties

    The Project Properties widget shows you important information about the selected project. Select the project you want to view from the widget menu .

    In this widget, you can view the following:

    • Created The date the project was created (e.g., added to Central)
    • Last Modified The date the project was last modified (e.g., changes were uploaded to Central, or changes made in Central)
    • Source Project Size The size of the source files of the project (Content and Project Folders)
    • Add Storage If you have a very large project, you may find yourself running out of space in Central. If you have the Server Management permission, click Add Storage to open the Manage License page, where you can purchase additional storage space. More information coming soon. xxx
    • Total Project Size The total size of all the source files and all builds
    • Project Size Graph A visual breakdown of how different file types use space in the project. The key below the graph shows how builds, resources, topic files, and project files use the total space.
    • Primary Target The output icon (e.g., PDF, HTML5) and name of the target.
    • Total Live Builds The number of builds set to "Live" for this project.
    • Total Targets The total number of targets in this project.
    • Total Builds The total number of builds in this project.

    See About Projects.

    If you have a Project Properties widget on your Project dashboard, you can copy it to the Home dashboard. See Duplicating Widgets.

    Data will not display in this widget until you select a project.

         Storage and Usage

    The Storage and Usage widget shows you important information about the MadCap Central license. This widget is visible to all users, but accessing information (e.g., purchasing additional seats) requires the Purchasing, User Management, or Server Management permissions.

    In this widget, you can view or do the following, depending on your permissions:

    • MadCap Central Renewal Date The date the MadCap Central license will need to be renewed. If you have the Purchasing permission, click Renew to renew the license. See Renewing a Central Subscription.
    • Used/Available Users The number of users currently in the system and the number of users allowed. If you have the User Management permission, click Add Users to open the Manage License page, where you can purchase additional user licenses. More information coming soon. xxx
    • Used/Available Storage The number of users currently in the system and the number of users allowed. If you have the Server Management permission, click Add Storage to open the Manage License page, where you can purchase additional storage space. More information coming soon. xxx

    This widget is only available on the Home page.

         Task Calendar

    The Task Calendar widget shows the tasks due, based on the filter settings you select from the widget menu (see Filtering Widgets).

    In this widget, you can view the following:

    • Calendar Corner marks on a date indicate that a task is due on that date. The color of each mark indicates the priority of the task due (red=high, yellow=medium, green=low). Click a date to filter the task list for that date. Use the arrows on the header bar to navigate between months.
    • Due Date In the task list, you can see the day a task is due.
    • Priority Level In the task list, you can see if a task is low, medium, or high priority.
    • Task Title Click the task title to open the task's details.

    See About Tasks.

    Data will not display in this widget until a task is added.

         Task Summary

    The Task Summary widget shows a doughnut chart that breaks down the percentage of tasks in each milestone (e.g., To Do, In Progress, Complete). You can also see a summary of each milestone and the number of tasks in each priority (e.g., Low, Medium, High).

    See About Tasks.

    Data will not display in this widget until a task is added.
  4. Click Add. The widgets are added to the dashboard.
 Note

If you archive a project or add an archived project to a widget, the widget will remain on the dashboard but its functionality is limited. You can still filter the widget but you will not be able to access the project. Archived projects' names are grayed-out in widgets.

 What's Next?

After you add a widget to a dashboard, you can use the widget menu to filter it to display relevant information. See Filtering Widgets.
