Importing from Perforce

This topic chapter section focuses on importing a project from source control. You might use this method, for example, if you are working on a multi-author project and another member of the team has placed the project in Perforce. For more information see About Source Control.

This topic focuses on importing a project from source control. You might use this method, for example, if you are working on a project with several other translators and another member of the team has placed the project in Perforce. For more information see About Source Control.

Following are the steps for importing a project from Perforce.

how to import a project from source control

    • Ribbon Select File>New Project>Import Project.

    • Source Control Explorer From the View ribbon, open the Source Control Explorer. From the drop-down, select the Home pane. Click Import Project.

    The Import Project from Source Control Wizard dialog opens.

  2. From the drop-down, select Perforce.
  6. Next to the Location field, click Browse. The Browse Depots and Streams dialog opens.
  7. Select a depot or stream (i.e., the name of your project).
  9. Click Next.

  10. Next to the Project file field, click Browse. The Browse Source Control Files dialog opens.

  11. Find and click on the project file (FLPRJ)(LIPRJ) that you want to import. (You may need to log in with your user name and password.)
  13. Click Next.
  14. In the Project name field, the name of the project being imported is displayed. It is recommended that you leave the name as it is, especially if you are working with other authors users on the project. However, you can enter a different project name if you want.
  15. In the Project folder field, either accept the default location for the new project or click to browse for and select a folder.
  16. Click Finish. The project is imported and loaded into .
