Binding a Project to Subversion

Use the following steps if you have already created a project and want to bind ("connect") it to Subversion. You can also automatically detect existing source control bindings if your project has been previously connected to Subversion.

how to bind a project to source control
  1. Open the project.
  2. In the Properties section select Project Properties. The Project Properties dialog opens.
  3. Select File>Project Properties.
  14. In the Project Properties dialog, click OK. The project is connected to Subversion, and you can now commit files as necessary.
 how to Bind a Project to source control—Source Control Explorer
  1. Open the project.
  2. In the Explorer section select Source Control Explorer. The Source Control Explorer opens.
  3. In the Pane section select Source Control Explorer. The Source Control Explorer opens.

  13. The project is connected to Subversion, and you can now commit files as necessary.
 Automatically Detecting Existing Project Bindings

You can scan a project when you open it to see if it has any bindings to source control. If a binding is detected, you can apply the binding and commit the project to source control.

 notes

You can also bind a new project to source control while creating it. See Creating a New Project.See Creating New Projects.

If you want to publish your output to source control, you must create a bind destination. See Creating Publishing Destinations.
