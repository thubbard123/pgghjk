Importing a Project from Microsoft Visual Studio Online

Importing from Microsoft Visual Studio Online

includes Microsoft Team Foundation Server support for Microsoft Visual Studio Online. If you manage your source control projects with Microsoft's online version of Visual Studio, you can import these projects to .

Use the following steps if you have already created a project and want to import it to from Microsoft Visual Studio Online.

Following are the steps for importing a project from Microsoft Visual Studio Online.

how to import a project from source control

    • Ribbon Select File>New Project>Import Project.

    • Source Control Explorer From the View ribbon, open the Source Control Explorer. From the drop-down, select the Home pane. Click Import Project.

    The Import Project from Source Control Wizard dialog opens.

  2. From the drop-down, select Microsoft Team Foundation Server.

  3. In the Server field, enter the URL of the project's "Default Collection."

    When binding to Visual Studio Online project using TFS, you must manually enter the URL in the Server field. You can only browse for a server location when binding to a local project using TFS.

    You may need to obtain this information from your system administrator. Click OK in the dialogs until you return to the main wizard page.

    If you run into issues in this dialog, chances are your user permissions do not have access to all of the collections on the TFS server. Please try to enter in the server/collection path manually.

  4. Next to the Team Project field, click . You may need to log in to Microsoft Visual Studio Online. If the Log In dialog opens, complete the User name and Password fields.

    The Browse Team Foundation Server dialog appears.

  5. Click on the Team Foundation project.
  7. Click Next.

  8. Next to the Project file field, click Browse. The Browse Source Control Files dialog opens.

  9. Find and click on the Flare project file (FLPRJ) that you want to import. (You may need to log in with your user name and password.)
  10. Find and click on the Lingo project file (LIPRJ) that you want to import. (You may need to log in with your user name and password.)
  12. Click Next.
  13. In the Project name field, the name of the project being imported is displayed. It is recommended that you leave the name as it is, especially if you are working with other authors users on the project. However, you can enter a different project name if you want.
  14. In the Project folder field, either accept the default location for the new project or click to browse for and select a folder.
  15. Click Finish. The project is imported and loaded into .
 What's Next?

After you import a Microsoft Visual Studio Online project, you can use the interface in the same way as you would with any other project bound to Microsoft Team Foundation Server.

You may still need to use Microsoft Visual Studio Online for some administrative tasks (in the same way as you might need to use Microsoft Visual Studio occasionally when binding a project to Team Foundation Server). For information on using Microsoft Visual Studio Online, see the Microsoft Visual Studio Online Help.
