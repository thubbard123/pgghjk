Previewing Topics

You can preview the topic based on the output format used in any of the targets in your project. You can also preview the output for a specific topic or using certain conditional expressions. The following steps show you how to do this.

how to preview topics
  1. Open the topic that you want to preview.

  2. To preview using the primary target, press CTRL+W.

  3. (Optional) When you preview for a specific target, the condition tags set for that target are automatically used in the preview. However, if you want to test the output with certain condition tags included or excluded, click in the upper-right corner of the dialog.

    The Conditional Text dialog opens. Either the Basic or Advanced mode is already selected, depending on which mode is being used in the target.

  4. You can use the same steps that are used in the target for including and excluding certain condition tags. For more information about the Basic and Advanced modes, see Basic Versus Advanced Conditions or Associating Condition Tags with Targets.For more information about the Basic and Advanced modes, see the online Help.

    Changes in the Preview window are not transferred to the target settings. They're simply temporary so that you can see what a topic looks like with certain condition settings.

  5. When you are finished, click OK.
  6. Click the X in the top-right corner of the Preview dialog to close it.
 how to preview topics
  1. Open the topic that you want to preview.

  2. Do one of the following:

    • Preview the Primary Target Press CTRL+W or click the face of the .

      You can change the target settings from the Preview window pane later, if desired.

    • Preview Another Target Click the down arrow next to the button, then select the target you want to preview.

    The Preview window pane opens using the target settings you selected.

  3. (Optional) If you want to preview a specific topic, or you want to change the target or conditional expressions applied to the current topic, do any of the following.

    How to Preview a Specific Topic

    • Next to the Topic field, click and select the topic you want to preview. The topic opens in the Preview window pane.

      Previewing a topic does not affect the topic open in the XML Editor. This allows you to preview a different topic from the one you are currently working on.

         How to Change the Target Settings Applied to the Preview

    You can change the target settings for your preview at any time.

    • From the Target drop-down, select the target settings you want to apply to the preview. When you select a target, the current preview changes accordingly, showing the content using the condition tag settings in that target.

         How to Change The Conditional Expression Settings Applied to the Preview

    When you preview for a specific target, the condition tags set for that target are automatically used in the preview. However, you can also preview your content with different condition tags included or excluded.

    1. Next to the Conditional Expression field, click . The Conditional Text dialog opens.

    2. Select the condition tags you want to include or exclude. Conditions already set in the target are set by default.

      Either the Basic or Advanced mode is already selected, depending on which mode is being used in the target. For more information about the Basic and Advanced modes, see Basic Versus Advanced Conditions or Associating Condition Tags with Targets.For more information about the Basic and Advanced modes, see the online Help.

      Changes in the Preview window are not transferred to the target settings. They're simply temporary so that you can see what a topic looks like with certain condition settings.

    3. When you are finished, click OK.
  4. (Optional) Click in the local toolbar to refresh the Preview window pane. This ensures you are seeing the most up-to-date content in the window pane.

  5. (Optional) Click to lock the Preview window pane. By enabling the lock button, you can continue to show the preview for one topic while opening other topics in the XML Editor.

  6. Click the X in the top-right corner of the Preview window pane to close it.
 Example note

The preview feature is useful because it is a quick way to see how a topic looks in the output. However, there may be times when you need to generate the entire target and test the output. For example, you might want to click hyperlinks or verify cross-references. See Testing Output.
