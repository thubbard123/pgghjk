About Dictionaries

Dictionaries are used in to support the spell checking feature. See About Spell Check.

Built-In Dictionaries

When you create a new project in , you select a target language (see About Language Support) (see Creating New Projects). Several of the languages listed in the wizard (i.e., those in bold) are associated with full dictionaries already installed for you.

These built-in dictionaries are automatically stored in the App Data folder on your computer, which can be accessed in various ways, depending on your operating system. For example, here is how you can open this folder in Windows 7:

  1. On your desktop, click the Start button.
  2. In the search field, type %appdata% and press Enter.
  3. Open the MadCap Software subfolder, then Dictionaries, and finally Installed.

You can also see which dictionaries are installed on your computer by opening the Options dialog in and selecting the Spelling tab. See .

Green dots indicate which dictionaries are supported by spell checkers and hyphenators. A hyphenator automatically inserts separators in an editor based on particular word patterns, as outlined in the dictionary file. Hyphenators are important only for certain languages.

For more information about hyphenation patterns in different languages, see http://hunspell.sourceforge.net/tb87nemeth.pdf.

 Custom Global and Project Dictionaries

In addition to all of the built-in dictionaries that are installed on your computer, you can develop custom dictionaries. You can think of these as supplements to the built-in dictionaries. A custom dictionary is created and modified automatically whenever you perform a spell check and tell that a questionable word is okay and should be added to a dictionary. See Spell Checking While Typing and Spell Checking Manually.

There are two kinds of custom dictionaries that you can use: global and project:

  • Global Dictionary

    See Choosing the Location for Global Dictionaries.

  • Project Dictionary A project dictionary is just what it sounds like. It is unique to a particular project. When a project dictionary is created, it is stored in the Advanced folder in the Project Organizer. When a project dictionary is created, it is saved in the Spelling folder of your Lingo project.

    For example, you might choose to use this kind of dictionary if you are a writer translator with several clients, and each client has unique terms with unusual spellings. By using a project dictionary, you can keep those spellings tied to a certain project.

Tasks for Using Dictionaries

Following are the primary tasks for using dictionaries:

  • Add Words to Custom Dictionary When performing spell checking in your project, you can add questionable words to a custom global or project dictionary. See Spell Checking While Typing and Spell Checking Manually.
  • Edit Custom Dictionary See Editing Custom Dictionaries.
  • Choose Global Dictionary Location Rather than using the default location, you can choose a custom location for global dictionaries. This is useful if you work on a team and want all writers to use the same dictionary. See Choosing the Location for Global Dictionaries.
  • Import If necessary, you can download more dictionaries in various languages from openoffice.org and then use the Options dialog to import them for use in . See Importing Dictionaries.
