Working with a Translation Project

After you have uploaded your project to a SharePoint Translation Library, translators can check the documents out of SharePoint and begin translating them.

When the Doc-To-Help project is opened, there will be a new Translation ribbon and window. You can use them to track the status of the documents as they go through the workflow and are translated, reviewed, and approved.

As the documents travel through the workflow, their status will be displayed under the Translation tab.

How to Open a Document

  1. Choose a document in the Documents pane or the Translation window.

  2. On the Translation tab, click the Open Server Copy or Open Local Copy button. The document opens in Microsoft Word.

    To open the original version of the document chosen, click the Open Original Document button.

 How to Open the SharePoint Translation Library
  • On the Translation tab, click the Open in Browser button.
 How to Set the Translation Status of a Document
  1. Choose a document in the Documents pane or the Translation window.
  2. On the Translation tab, click the Translation Status button.
  3. Select a status:
    • Not Started
    • In Progress
    • Completed
 How to Refresh the SharePoint Translation Library
  • On the Translation tab, click the Refresh button.  Updated statuses will display under the Translations tab.
Notes

This is supported in SharePoint 2010 only. Translation Libraries are not included in SharePoint 2013.

In this process, your project documents are living ones. If you check out documents and edit them (see Checking Documents In and Out of the SharePoint Library), the translators will need to compare their documents to the edited ones and adjust accordingly. See Merging Word Documents.

You can synchronize your documents with the SharePoint server, and work offline. See Synchronizing Documents , and Working Offline .

If you need to add a new document to any existing translation project, it must be added to the appropriate SharePoint Translation Library first. Then click the Add New Documents button in the Documents pane and choose SharePoint for document location. See Adding SharePoint Library Documents to a Doc-To-Help Project .

When document translation is complete for the project, you can build it in Doc-To-Help. If you wish to localize the theme, you can translate it (see Customizing Themes). You can also change other project settings, such as the name of the project and other text strings in the Help Targets dialog (see Creating and Deleting Targets).
