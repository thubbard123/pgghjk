Accessibility Standards and Guidelines

accessibility standards and guidelines alt and title attributes (alternative text and screen tips) images—apply alternate text and screen tip to all references

When you insert or edit an image, you will see a check box that allows you to apply the same alternate text and screen tip for all instances of the image throughout your project. See Inserting Images and Editing Images.

 images—using empty alt text

See Applying Empty Alt Text and Inserting Images.

 different outputs for different users

One way to make content accessible is to provide one output for most users and an entirely separate output for individuals with disabilities. In the locations where you have elements that can present accessibility issues (e.g., images, multimedia), you can provide descriptions of those elements. Then by using condition tags and different targets, you can easily have customized outputs for each set of users. See Determining the Output Type, About Condition Tags, Adding Targets, and Editing Target Settings.

 semantic style elements

You can use character-level styles when formatting text. For example, the <strong> tag conveys the idea that you want to emphasize the importance of some words in a paragraph. See About Styles and Stylesheets.

 tables

You can add captions, summaries, and header rows on tables. See Inserting Tables and Editing Table Properties.

 analyzer scans

You can use MadCap Analyzer to scan your project for areas where you can improve accessibility. See About Analyzer and Enabling Analyzer Scans.

 html5 and webhelp outputs pdf output eclipse help output accessibility warnings notes

If you also have the latest version of MadCap Analyzer installed, you can see accessibility warnings before you generate output.

In addition to the tasks described, which can be performed in the user interface, you can accomplish a multitude of other tasks by working directly in the XHTML markup (via the Internal Text Editor or another editor such as Windows Notepad). For example, you can add labels on forms or titles on frame elements that you create. The w3.org website in particular provides a great deal of additional information and techniques for making documentation accessible by editing the markup. See http://www.w3.org/TR/WCAG20-TECHS/.

If an individual is using JAWS to access your Help system, that person needs to be in Forms mode in JAWS in order to navigate successfully in the output.
