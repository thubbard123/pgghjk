 How to Schedule a Target to be Generated
  4. In the toolbar, click . The Build Schedule dialog opens.

  5. Complete the fields in the dialog:

    Field

    Description

    24h time

    From the first drop-down, choose the hour at which you want to the build to start, and from the second drop-down choose the minute. A 24-hour clock is used.

    For example, to build at 3:10 a.m., you would choose 3 from the first field and 10 from the second field. To build at 7:45 p.m., you would choose 19 from the first field and 45 from the second field.

    In the next drop-down field, choose your time zone.

    Su M Tu W Th F Sa

    Click on the day(s) of the week when you want the target to be generated.

    Build Even if Nothing Changed Since the Previous Build

    Select this option to always generate output at the specified days and times, even if no source files were changed. If you do not select this option, the target will be generated only if a change is detected in the source files.

 What's Next?

After you generate a target, you might want to do any of the following:

  • View You can open the output to see how it looks. See Viewing Output.
  • Open Log If the build process shows warnings or errors, you can open the build log to see more details. See Opening the Build Log.
  • Set “Live” or “Keep” If you want to make a particular build accessible to your end users, you can set it as “Live.” You can also set a build to “Keep,” which means that you do not want it to be deleted; rather, you might want to set it to “Live” later on. See Setting Builds to Live or Keep.
