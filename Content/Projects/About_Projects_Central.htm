 Project Cards

Any user can hover over a project in the grid to display the project card.

Depending on your association with the project or your permission settings, you can do the following from the card:

  • View Profile See Viewing Project Profiles.
  • Edit Profile If you have “Manage Teams/Projects” permission, you can edit the profile for a project. This includes the ability to change the project’s name, initials, description, and associated color. You can also change which teams and/or users are associated with the project. See Editing Project Profiles.
  • Open Dashboard See Opening a Project.
  • Open Build Management View See About Builds.
  • View Activity You can open the Recent Activity dialog to see what has been going on with a given project. See Viewing Project Activity.

 Projects and Source Control in Central

Basic Source Control Activities

Basic Source Control Activities

Following are the main activities related to source control in Central:

  • Upload/Import As already mentioned, you can upload (bind) Flare projects to Central (see Uploading (Binding) Projects) and import projects from Central to your computer (see Importing Projects). These are two of the most important tasks related to source control when it comes to Central.
  • Push Another source control task that is essential—whether you are working in the dual-bound model or the single-bound model—is that of pushing files. See Pushing Content.
  • Pull If you are working in the single-bound model with other writers, you will probably also need to pull content from Central to your local Flare project. This can be done from the Source Control ribbon in Flare, and ensures that you have the latest files locally, in case other users have made changes. See Pulling Content.
