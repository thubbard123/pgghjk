Pulling Content

If you are working in a single-bound model as opposed to a dual-bound model, you can pull—or download—content from Central (see Single-bound Projects and Dual-bound Projectssee Single-bound Projects and Dual-bound Projects). A lone writer would probably not need to do this, but if there are other writers on your team authoring in the same Flare project, you will periodically need to pull from Central to make sure your local files are in sync with those on Central.

If you are working in a dual-bound model, you cannot pull files from Central (unless you are bound to Git; see the note below). Instead, you would pull files from your other third-party binding using Flare's built in source control system (for more information, see the Flare online Help). Once your files are up to date with that binding, you would push your files to Central. See Pushing Content.

Note How to Pull Content in a Single-bound Project
  1. Open the project in MadCap Flare.
    • Ribbon In the Project section select Pull.
    • Right-Click If you have the Content Explorer, Project Organizer, Pending Changes window pane, or File List open, right-click on any file and select Source Control>Project>Pull.
  5. The Select Remote for Pull dialog opens. From the Remote drop-down, select MadCap Central.
