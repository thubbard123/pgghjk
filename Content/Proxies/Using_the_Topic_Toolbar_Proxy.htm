Creating a Topic Toolbar

basics of using topic toolbars

The generation of topic toolbars is based on the following elements:

  • Topic Toolbar Proxy

  • Skin The way the skin works with topic toolbars is a bit different for HTML5 than it is for outputs that are using Standard and Mobile skins.

    • HTML5 Skins Although topic toolbars can be inserted into most online outputs, only HTML5 uses a Topic Toolbar skin component as part of the process. With HTML5, the settings in the Topic Toolbar skin component have precedence over any settings in the Topic Toolbar Proxy dialog, including the selection of which buttons to be included in the toolbar. See Topic Toolbars in HTML5 Skin Components.

    • Standard and Mobile Skins See Specifying Topic Toolbar Settings in a Skin—WebHelp Outputs and Specifying Topic Toolbar Settings in a Skin—WebHelp Mobile.

      See Standard Skin Styles and WebHelp Mobile Skins.

  • Style Class Regardless of the type of output, you also have the option of using the Madcap|topicToolbarProxy style to change the look of the toolbar. See Using Styles to Determine the Look of Topic Toolbars.

If you are generating an target that uses a Topic Toolbar proxy, Flare will automatically remove the proxy when building the output. This is because the Eclipse development platform provides its own toolbar support. See the Platform Plug-in Developer Guide from Eclipse at: http://help.eclipse.org/kepler/index.jsp

 examples how to insert a topic toolbar

Following are steps for inserting a Topic Toolbar proxy. Again, for most of the supported outputs, you can select the buttons either in these steps or in the skin.

    • Open a topic where you want the toolbar to appear. This is a good method if you only want the toolbar to be shown in one topic, or just a few topics.

      OR

    • Open a master page that you will use when generating the target. This is the best method if you want the toolbar to be shown on all topics in the output.
  2. Place your cursor where you want the toolbar to be shown in the output (e.g., bottom or top of the topic or master page). If you are working in a master page, you may need to press the down arrow on your keyboard and then press Enter at the correct location.
  3. Select Insert>Proxy>Insert Topic Toolbar Proxy. The Topic Toolbar Proxy dialog opens.

  4. (Optional) If you are working with HTML5 output, you can select a skin component in the Skin File field. From the Skin Editor, you can select the buttons to be included in the toolbar (see Editing Skin Settings and HTML5 Skin Styles). Alternatively, you can select buttons from the Buttons field below.

    If you do not select a skin component in this field, Flare uses the first one it finds in your project (if one exists). Otherwise, Flare provides a default design.

  5. (Optional) In the Stylesheet class for proxy field, you can select a class to affect the look of the entire toolbar.

    You might create and use a proxy style class, for example, if you want to add a border around the toolbar. If you do not select a class from this field, the generated toolbar will use the style settings from the parent MadCap|topicToolbarProxy style. You have the option of creating a class for this proxy style in the Stylesheet Editor. To do this, select the MadCap|topicToolbarProxy style and click Add Class to create a class. The class will then be available from this field.

  6. (Optional) You can select buttons for the toolbar here, rather than doing so in the skin.

 Editing a Topic Toolbar

After inserting a Topic Toolbar proxy, you can edit it. To do this, open the topic or master page where the proxy is inserted, right-click on it, and select Edit Topic Toolbar Proxy.
