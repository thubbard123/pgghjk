Creating Styles

You are almost finished. The page layouts, topics, and TOC have been created. And you've added all of the content to the topics. Most of the content is already using the appropriate styles, but there are a few pieces in the topics that still need to be formatted.

All of the styles you need already existing in the stylesheet in the template. But we're going to show you how we created a few of the styles so you can see how it is done.

Identifying Styles

Identifying STyles

Before we create new styles, we want to show you how to identify existing styles. If you've ever looked at a section of content and thought, "I wonder how they got it to look like that," you can easily find out. It's just a matter of knowing where to look.

  1. Open the Back2.htm topic.
  2. In the third panel, locate the bulleted list and click in it somewhere.

  3. Look to the far left side of the XML Editor and notice the structure bars. If your structure bars are not turned on, click at the bottom of the editor.

    The structure bars that are related to the list are those with the ul and li labels. The ul structure bar represents the list as a whole, and each li bar represents an item in that list.

  4. Now look above the topic. You should see a span bar. If your span bars are not turned on, click at the bottom of the editor.

    Span bars represent formatting that is done inline. In our list, we have some text at the beginning of each item that is shown in light green. Each of these is using a span style.

  5. Let's investigate the ul and li styles first. Right-click the ul structure bar and select Style Class>Edit Style Class.

    The stylesheet opens and automatically selects the style class. In this case, the style class is named "ul.DarkBackground."

  7. In the local toolbar, click . This changes the view of the properties on the right from groups to an alphabetical list.

  8. On the right side of the local toolbar, click the Show drop-down field and select Show: Set Properties. This filters the properties below to only those that have been explicitly set in the stylesheet for the selected style, as well as any properties explicitly set in an inherited stylesheet.

    As you can see, only two properties were set on this style. One points to a small image that serves as the bullet. The other property sets the left margin at 60 pixels.

    But there is one other style to look at. Notice that the text is very light so that it shows up on the dark background. This was caused by a generic style class, which is a class that is not associated with any parent style.

    In this example, the class we've looked at is called "ul.DarkBackground." A generic class with the same name (.DarkBackground) also exists in the stylesheet.

  9. In the list of styles, expand (Generic Classes), and click DarkBackground. The property set for that class (a light gray color) is shown to the right.

  10. Scroll down the list of styles until you find li, and click on it. The properties to the right show what has been set for the individual list items.

  11. Now you know how the list was designed. So let's take a look at the inline formatting that created the light green text at the beginning of the list items. Go back to the Back2.htm topic.

  12. With your cursor still placed in the list, right-click the span bar above the topic, and from the context menu select Style Class>Edit Style Class.

    The stylesheet opens with the relevant style class selected. In this case, it is a class called "span.Light." And its property settings are shown to the right. You can see that it has a light green color and the font is bold.

    You can use these steps to identify settings for any styles you find in the Front and Back topics.

Creating Styles

creating Styles

Let's re-create a couple of the style classes that are used in this template. If you look at the original Front topic, you will see that the first two paragraphs (which have variables) are designed to look differently than a normal paragraph. There is more space above the first paragraph, and the second paragraph is larger with a green font.

We are going to create duplicate style classes that we will apply to our front topic.

  2. On the left side of the Stylesheet Editor, select the p style.

  3. In the local toolbar, click Add Selector.

  4. In the New Selector dialog, click in the Class Name field and type the following:

    ManualTitle1New

  5. The new class is added under the p style and is selected.

  6. On the right side of the local toolbar, click in the Show drop-down field and select Show: All Properties. This displays all of the available properties below.

  7. Scroll down the list of properties until you find each of the following properties, and give them the values shown.
    • column-break-before always
    • font-family Century Gothic
    • font-size 9pt
    • margin-bottom 0
    • margin-left 50px
    • padding-top 80px
  9. In the local toolbar, click Add Selector.

  10. In the New Selector dialog, click in the Class Name field and type the following:

    ManualTitle2New

  11. The new class is added under the p style and is selected.
  12. In the list of properties, set the following for the new style class:
    • color #bed230
    • font-family Century Gothic
    • font-size 22pt
    • line-height 0
    • margin-bottom 0
    • margin-left 50px
    • margin-top 0
