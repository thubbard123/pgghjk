Creating Page Layouts

Page layouts are used to configure the pages for print-based output and are especially important in this particular template because of its unique characteristics (e.g., multiple columns, images in the middle of columns, color and images that bleed off the page). This template comprises only two topics—one for the front of the product foldout and the other for the back of the foldout. Therefore two page layouts are required—one to be associated with each topic.

When you go through the following steps, you will see that two page layouts already exist in the project, and they're already completely edited. If you want, you can open these files and take a look around to see what you will be producing in the end. You can also build one of the targets in the project to see how these page layouts are incorporated into the output.

In the following sets of steps, we will create two new page layouts, replicating the look and substance of the existing page layouts.

Front Page Layout

front page layout

The existing front page layout in the project is simply named "Front." We'll give our new page layout a slightly different name.

Create a Page Layout File

  1. In the Content Explorer, expand the Resources folder, then expand the PageLayouts folder. Notice the existing two page layouts (Back and Front).

  2. Right-click the PageLayouts folder, and from the context menu select New>Page Layout.

    The Add File dialog opens.

  3. Under the Factory Templates, select the Default template, because it is quite simple and requires fewer changes than other templates.

  4. In the File Name field at the bottom of the dialog, type Front2 as the name for the new page layout.

  5. Click Add. The page layout is added to the Content Explorer, and the Page Layout Editor opens to the right, displaying the page layout.

  6. Before we continue with the rest of the steps for editing the page layout, let's make one quick change in it.

  7. Click the header frame (the rectangle) at the top of the page layout and press Delete on your keyboard.

 Set page size and margins

Now that we have created our first page layout, we will start making changes to it. To begin, we will adjust the page size and margins.

  1. In the local toolbar of the Page Layout Editor, click (click the face of the button, not the down arrow).

    The Page Properties dialog opens.

  2. Select the General tab.
  3. Click the Page Size drop-down and select (custom).
  4. In the Custom Width field, enter 15, and for the unit of measurement select Inch (in).
  5. In the Custom Height field, enter 5, and for the unit of measurement select Inch (in).
  6. Keep the Orientation field set to Portrait.

  7. In all of the Margins fields, enter 0.

    You can set some value for the margins on your page if you want. However, in our template we chose to set them all to 0 and specify the margins on our CSS styles instead.

  10. Because the view of the page layout is so large, it is difficult to see what's going on so far. Therefore, in the local toolbar, click View and select Fit Width. The page layout is zoomed out so you can see all of it.

For more information about specifying the page size and margins, see the online Help.

For more information about specifying the page size and margins, see Specifying Page Size and Orientation and Specifying Page Margins.

 Create Panels—body frame with columns

Now that we've resized the page layout, we're going to divide it into three equal panels, which will serve as the natural locations where the printed output can be folded. We will do this by creating columns in our body frame.

  1. In the Page Layout Editor, double-click the body frame.

    The Frame Properties dialog opens.

    An alternative is to simply click and drag the body frame, and its edges, until you have it placed just where you want it on the page. However, in this exercise, we'll use the Frame Properties dialog, because it's easier to get the precise size and position that you see in the template.

  2. Select the Frame tab.
  3. We want to set the size and position of the body frame so that it fits exactly on top of the page. So in the Size and Position section, click in the Width field, enter 15, and for the unit of measurement select Inch (in).
  4. In the Height field, enter 5, and for the unit of measurement select Inch (in).

  5. In the Padding section, click in the Left field, enter 10, and for the unit of measurement select Pixel (px).

    This is optional; you can set it to 0 if you like.

  6. In the rest of the fields on the tab, enter 0.

  7. Select the Columns tab.
  8. In the Count field, enter 3.

  9. In the Gap field, enter 0.

  11. The page layout should now look something like this:

 Set bleed Size

Next, we're going to set the bleed size. The bleed is the area beyond the edge of the page where printing occurs. Professional printers use the bleed to prevent a final trimmed document from having unprinted areas along the edge of the document.

  1. In the local toolbar of the Page Layout Editor, select View>Bleed. Now you can see the bleed area in the editor.

  2. In the local toolbar of the Page Layout Editor, click (select the face of the button, not the down arrow).

    The Page Properties dialog opens.

  3. Select the Print Marks tab. This lets us set the size of the bleed area.

  4. In all of the fields, enter 0.25 and for the unit of measurement select Inch (in).

    You can certainly keep the default setting of 0.50 in. We simply changed it in our template because 0.25 in is the industry standard. The most important thing is to make sure the bleed area is large enough. Ideally, it should be no less than 0.125 in.

  6. The page layout should now look something like this:

  2. Select the Frame tab.

  5. Click in the Height field, enter 44, and for the unit of measurement select Pixel (px).
  6. In the Anchors section, click in the Bottom field, enter 12, and for the unit of measurement select Pixel (px).

  7. In the rest of the fields on the tab, enter 0.

  8. In the lower-right corner of the tab, click the left, right, and bottom arrow so that it looks like this:

  9. Select the Alignment tab.

  10. Set the Vertical Alignment to Middle.

    This is purely optional. You do not need to specify how content is aligned in the frame. In our template, we decided to align the content in the middle of the frame vertically.

  12. The footer frame should now be repositioned and selected on the page.

  15. Select the Table ribbon.

  16. On the far left of the ribbon, click the face of the Insert Table button.

    The Insert Table dialog opens.

  17. Select the General tab.
  18. In the Number of columns field, enter 3.
  19. In the Number of rows field, enter 1.
  20. In the Number of header rows and Number of footer rows field, enter 0.

  21. In the AutoFit Behavior section, select Fixed column width, and in the field to the right select 33.333% (an equal percentage for each of the three columns in the table).

  23. Add content to the first column of the table:

    1. Insert three circle images in a row, each separated by a space (in our template, the image is named "FooterCircleBlack"). See Inserting Images. For more about inserting images, see the online Help.

    2. Resize the images you inserted. In the existing page layout, we resized ours to a maximum width of 30px. You can do this by applying a style to the images, or you can right-click on each image, choose Image Properties, select the Size tab, and edit the Maximum Size Width. See Resizing Images. For more about resizing images, see the online Help.

    3. Apply a style to the paragraph in that cell. To do this, click in that cell, select the Table ribbon, and in the Cells section, click Cell Content Style.

      Then in the Table Cell Content Style dialog, select p.FrameFooterImagesRight. This is a style class we created in the template to align the decoration images to the right. We will cover styles in more detail in a later lesson (see Creating Styles). Click OK.

      It should now look like this:

  24. Add content to the second column of the table:
    1. Type the number 2 to represent page 2. In our template, we did not use a page variable—like we would normally do in a page layout—because we're dealing with columns within a single page, rather than one page number per page.  Set Background Image

The front page layout is almost finished, but we want to add a background image to it. Flare allows you to add one background image to the entire page layout and one background image in each body frame. We're going to add one image to the background of the page layout in order to use it as a watermark, appearing behind all of the content added in the topic.

Another thing to keep in mind with page layouts is the dots per inch (DPI) of images. For print-based output, you ideally want images with high DPI settings. A DPI of 300 is recommended if possible, but sometimes you need to use an image with a lower DPI so that you can display it at the right size. The best practice is to start out with a very large image with a high DPI. When it is added to the Page Layout Editor, it appears to shrink because you've told Flare to display it at 300 DPI (or another high DPI setting). The larger the DPI, the smaller it's going to be displayed in the Page Layout Editor and in the output.

  1. In the local toolbar of the Page Layout Editor, click (click the face of the button, not the down arrow). The Page Properties dialog opens.
  2. Select the Background tab.
  3. In the Image section, click Browse.

  4. Under the Resources>Images folder, select Image1.eps. Click OK when you are done.

  5. In the Repeat field, select No repeat.

  6. In the Position section, click in the X field and enter -200px. This positions the image 200 pixels left horizontally.

    It's not uncommon to play with different values in the Position fields until you get the image placed just where you want it.

  7. Because this image was set in the background of the entire page layout, and the image is larger than the page, it bleeds beyond the edge of the page. If instead we had set the image in the background of a frame, it would stop at the edge of that frame.

  8. It should now look like this:

 Create decoration frame

For the second body frame in the back page layout, we have a background color. Because we anticipate this being printed on white paper, we need to make sure the color extends into the bleed area.

One solution is to click and drag the body frame so that it extends into the bleed area. But that means adjusting our styles to prevent topic text from moving into the bleed area.

Instead, we are going to create a decoration frame with the same background color. We will position this decoration frame around (and behind) the second body frame.

  1. In the local toolbar of the Page Layout Editor, click the down arrow next to the New Frame button and select New Decoration Frame Mode.

  2. Click on the page and draw a rectangle. It doesn't matter how big it is or where you draw it.
  3. Double-click that frame. The Frame Properties dialog opens.
  4. Select the Frame tab.

  5. Set the fields as follows:

    size and position

    • Left 480 px
    • Top -0.25 in
    • Width 10.25 in
    • Height 5.50 in

    anchors

    • Right -0.25 in
    • Bottom -0.25 in

    padding

    • Left 0
    • Right 0
    • Top 0
    • Bottom 0

  6. Select the Background tab.
  7. In the Color section, select Solid.
  8. In the Fill Start field, click the down arrow and select More Colors
  9. In the Color Picker dialog, highlight the number the # field and replace it with 4E4D4E (this matches the background of the second body frame).
  10. In the Color Picker dialog, click OK.
  11. In the Frame Properties dialog, click OK.
  12. Now we just need to make sure that the decoration frame is behind the body frame. So click on the decoration frame in the Page Layout Editor.

  13. In the local toolbar, select Layout>Depth>Sink To Bottom.

  14. The layout should now look like this:
