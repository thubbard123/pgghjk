Properties Dialog (Style Classes)

This dialog is used to apply different kinds of settings to styles, using various tabs. The tabs that display in the dialog depend on which type of style you have selected. See About Styles and Stylesheets and Editing Styles in a Regular Stylesheet.

Following are the primary tasks that you can perform in this dialog, depending on the type of style you have selected.

For more information see the Flare online Help.

Tasks Associated with Style Properties

  • Autonumber Formats Read more… Autonumbering can be used when you want to create content where certain portions are numbered automatically. When you incorporate autonumbering into content, you do so by creating an autonumbering format, which consists of one or more commands. Some examples of commands are: CH:, {n+}, {chapnum}, {b}, and {/b}. In addition, you can add text next to commands. See About Autonumbers and Creating Autonumber Formats for Styles.

  • Background Read more… See Setting the Background for Paragraphs.
  • Breaks—Column and Page Read more… See Setting Page and Column Breaks.
  • Borders Read more… You can add borders around different elements, such as paragraphs and images. Borders can be added on any side of an element (left, right, top, bottom), or all around it. See Setting Borders for Paragraphs and Adding Borders to Images.

  • Cross-reference Formats Read more… You can change the cross-reference format used by a style. Cross-references let you create automated links that are based on commands you provide. This allows you to keep links consistent and change them in just one place by using the MadCap|xref style. See About Cross-References and Editing Cross-Reference Style Formats. In addition, although users can click hyperlinks if viewing your output on screen, cross-references are even more useful for print-based output. The reason for this is that you can create commands where cross-references are translated to page numbers. For example, a link can be displayed as "See page 36" in print-based output, rather than merely "See My Topic."

    example

    example

    Here is an example of a cross-reference format, using a combination of text and commands:

    See {b}"{paratext}"{/b} on page {page}.

    A format such as this might be translated in the output as something like this:

    See "Dogs and Cats" on page 93.
  • Font Settings Read more… You can modify various properties for fonts in a style. This includes changing the font family, size, style, and color. See Editing Font Properties.
  • Footnotes Read more… After inserting a footnote, you can edit the properties for the footnote number and the accompanying comment in many ways through the use of styles. See About Footnotes and Editing Footnotes.
  • Hyphenation Read more… See Setting Hyphenation.
  • Images—Size Read more… You can resize many images at once by using the img style in the Stylesheet Editor. Not only can you select a specific image size, but you can also set a maximum or minimum height or width. You can set either the width or height on the image tag and let set the other property automatically, maintaining the aspect ratio (i.e., size proportion of the image). See Resizing Images.
  • Italic Correction Read more… See Setting Italic Correction.
  • Leader settings - for generated TOCs and lists Read more… For print-based output youYou can change the format of the area between headings and page numbers (i.e., the leader, which is often a series of dots) in a generated table of contents or list of elements. See Using Styles to Determine the Look of a Print TOC and Using Styles to Determine the Look of a List of Elements.
  • Letter Spacing Read more… See Setting Letter Spacing.
  • Object and Paragraph Positioning Read more… After you insert an object (such as an image or text box) into a topic, you can adjust its positioning on the page. You can also position paragraphs. For example, you can "float" objects and paragraphs so that they are displayed outside the frame that holds the normal flow of topic text. See About Positioning Elements and Using Floats for Positioning Elements.
  • Paragraphs—Alignment Read more… See Aligning Paragraphs.
  • Paragraphs—Indentation Read more… See Indenting Paragraphs.
  • Short Line Elimination Read more… See Setting Short Line Elimination.
  • Spacing Before and After Read more… See Setting Paragraph Spacing Above and Below.
  • Thumbnail Read more… See Showing Images as Thumbnails in Output.

  • Widows and Orphans Read more… See About Widows and Orphans and Setting Widow and Orphan Control.

    • Widow Control Read more… Determines the minimum number of lines in a paragraph to be left at the top of a page or column.
    • Orphan Control Read more… Determines the minimum number of lines in a paragraph to be left at the bottom of a page or column.
