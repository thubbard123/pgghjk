Project Organizer

The Project Organizer (along with the Content Explorer) is one of the most-used elements in the interface.

Just as it sounds, the Project Organizer contains and organizes all of your project-related files.

What You Can Do in the Project Organizer

what you can do in the project organizer

Following are the primary tasks that you can perform in the Project Organizer:

  • Auto Suggestion List Files (Add/Open) Read more… When you first start typing content in a topic, a popup may display below your text as you type. This is the Auto Suggestion feature, which analyzes text as you type and compares it to other content (e.g., phrases, variables, snippets) that already exists in the project. This means you can quickly select existing content from the popup so that you do not have to finish typing it. See About Auto Suggestion and Adding Auto Suggestion List Files.
  • Browse Sequences (Add/Open) Read more… If you have several topics that you think end users should read in order, you can create browse sequences. When users view the compiled online Help system, they can use your browse sequences to "browse" through topics in a particular order. See About Browse Sequences, Adding a Browse Sequence, and Opening a Browse Sequence.
  • Conditions (Add/Open/Apply) Read more… A condition tag set is used to hold condition tags you create for your project. You can create as many condition tags as you want for a condition tag set. If you want more condition tag sets to hold even more condition tags, you can easily add them. You can also expand the condition tag sets in the Project Organizer to see the conditions within them. If you want to apply a condition to content, you can click and drag it from the Project Organizer to the content in the XML Editor. See Adding Condition Tag Sets, Opening Condition Tag Sets, and Applying Condition Tags to Content.
  • Context-sensitive Help—Alias Files (Add/Open) Read more… An alias file is used to populate a header file with the information necessary for producing context-sensitive Help (CSH). In , you can open an alias file and use the Alias Editor to create and assign identifiers for the header file. See About Context-sensitive Help, Adding Alias Files, and Opening Alias Files.
  • Context-sensitive Help—Header Files (Add/Open/Export) Read more… A header file is a simple text file that contains basic information about connecting the dialogs or windows in a software application to the corresponding topics in the Help system. Both you and the software developer need access to this file. See About Context-sensitive Help,Adding Header Files, Opening Header Files, and Providing a Developer with a Header File.
  • DITA—Import File (Add/Open) Read more… A DITA import file is used to import DITA file content into a project. See Adding a DITA Import File and Opening a DITA Import File.
  • Export File (Add/Open) Read more… An export file is used to export an entire Flare project, or parts of one, to another location. See Adding an Export Project File and Exporting Projects.
  • File Tag Sets (Add/Open) Read more… A file tag set is used to assign "tags" to topics and any other files in , even folders. You can use file tags for many different purposes, such as assigning authors or milestones to topics. lets you generate reports based on the tags that are assigned. This makes project development easier to track, manage, and schedule. See Adding File Tag Sets.
  • Flare Project—Import File (Add/Open) Read more… A project import file is used to import files from other projects. See Adding Project Import Files and Global Project Linking—Importing Files from Other Projects.
  • Folders (Create) Read more… When you add project files, they are added to various folders in the Project Organizer. As your list of files grows, it may become somewhat difficult to find a specific file when you want to work on it. In order to keep your files organized and easier to find, you can create subfolders in the Project Organizer and move files into them. See Creating Subfolders.
  • FrameMaker—Import File (Add/Open) Read more… A FrameMaker import file is used to import FrameMaker documents into a project. See Adding a FrameMaker Import File and Opening a FrameMaker Import File.
  • Glossaries (Add/Open) Read more… A glossary is a feature that you can add to your output to help users understand the meaning of individual terms. See About Glossaries, Adding a Glossary, and Opening a Glossary.
  • Indexes—Auto-indexes (Add/Open) Read more… An auto-index lets you automatically add words and phrases in your project to a generated index, rather than inserting the index markers manually. In the Project Organizer, you can add and open an auto-index phrase set. See Creating Auto-Indexes.
  • Indexes—Link Sets (Add/Open) Read more… This is an index entry that links to another index entry. There are three kinds of index links: "See," "See Also," and "Sort As." In the Project Organizer, you can add and open an index link set. See Creating Index Links.
  • Language Skins (Add/Open) Read more… See Creating User-defined Language Skins and Editing Language Skins.
  • Publishing Destinations (Add/Open) Read more… A publishing destination is a project file that contains the information needed for publishing output (i.e., copying the files) to another location (such as a website or a directory on your computer). See Adding Destinations and Opening Destinations.
  • Relationship Tables (Add/Open) Read more…  See About Relationship Tables, Adding Relationship Tables, and Opening Relationship Tables.
  • Reports—Analyzer (Add/Open) Read more… In you can  See About Analyzer. Also see About Reports.
  • Search—Filter Sets (Add/Open) Read more… A search filter lets users narrow their search based on concepts that you have inserted into topics. Concepts are simply keyword markers that you insert into topics that have some kind of relationship with each other. They are also used for inserting concept links into topics (such as the "See Also" link that you see at the bottom of this topic). See About Search, Adding Search Filter Sets to Projects, and Opening a Search Filter Set.
  • Search—Synonym Files (Add/Open) Read more… A synonym file lets you create synonyms for terms in your project so that users are more likely to find the information they need when performing searches. See About Search, Adding Synonym Files, and Opening Synonym Files.
  • Skins (Add/Open) Read more… A skin is a file that contains information about the appearance of the output window. See About Skins, Adding Skins to a Project, and Opening Skins.
  • Tables of Contents (Add/Open) Read more… In many cases, provides you with an initial table of contents (TOC), which you further "build" (or create) using the TOC Editor. You can use this TOC (or another one) as your primary, or "master," TOC. At some point, you may decide to add another TOC to the project. The extra TOCs that you add can then be linked to the master TOC. See About Tables of Contents, Adding a Table of Contents , and Opening a Table of Contents.
  • Targets (Add/Open) Read more… A target is one "instance" of an output type. When you build your final output, you are essentially building one or more of the targets in your project. You can add regular targets, as well as batch targets. See Adding Targets, Building Output Using a Batch Target, and Opening Targets.
  • Variables (Add/Open/Insert) Read more… A variable set is an element used to hold one or more variables. A variable is a pre-set term or content that you can use in your project over and over. Variables are similar to snippets. You can expand the variable sets in the Project Organizer to see the variables within them. If you want to insert a variable into a document, you can click and drag it from the Project Organizer to the XML Editor. See About Variables, Adding Variable Sets to Projects, Opening a Variable Set, and Inserting Variables.
  • Word—Import File (Add/Open) Read more… A Word import file is used to import Microsoft Word documents into a project. See Adding an MS Word Import File and Opening an MS Word Import File.
note

You can also right-click on files in the Project Organizer and perform tasks from the context menu that displays. For example, you can select "Open with" and open the selected file in another application (e.g., Notepad).

Buttons and Sections

Following are explanations of the different buttons and sections in the Project Organizer:

Button/Section

Description

Toggles between showing or hiding the folders in the Project Organizer.

Splits the Project Organizer into two halves. The right half shows additional details for the files. When you click a file on the right side, a red circle shows the location in the Project Organizer.

Advanced

Holds the various types of files after you add them to the project. These include alias files, auto-index phrase sets, Auto Suggestion list files, browse sequences, file tag sets, header files, index link sets, relationship tables, search filter sets, and synonym files.

Conditional Text

Holds the condition tag sets that you add to the project. You can expand the condition tag sets to see the conditions within them, and you can drag those conditions to content in the XML Editor.

Destinations

Holds the publishing destinations that you add to the project. Depending on the project template you select, may provide you with an initial destination called "MyDestination."

Glossaries

Holds the glossaries that you add to the project. provides you with an initial glossary called "MyGlossary."

Exports

Holds export files that you add to the project. These are used to export your Flare project (or parts of it) to another location.

Imports

Holds import files that you add to the project. This can include project files, files, files, and files. The import files have an .flimpfl extension, the Word import files have an .flimp extension, the FrameMaker import files have an .flimpfm extension, and the DITA import files have an .flimpdita extension.

Reports

Holds Analyzer reports that you create in order to track information in your project. It also holds error reports that you save after building output.

Skins

Holds the skins that you add to the project. There are three types of skins—Standard, HTML5, and WebHelp Mobile.

Targets

Holds the targets that you add to the project. After you create a new project, you will have only one target in this folder, based on your selection in the wizard. However, you can add as many targets to a project as you need. In addition to regular targets, you can add batch targets, which are used to generate multiple targets in one batch.

TOCs

Holds the tables of contents that you add to the project. If a TOC has a green arrow and the word "(Master)" after it, that means it is designated as the master TOC. You can choose another TOC to be the master by right-clicking on it and selecting Make Master TOC from the context menu.

Variables

Holds the variable sets that you add to the project. You can expand the variable sets to see the variables within them, and you can drag those variables to content in the XML Editor.

[Preview Bar]

This is the area at the bottom of the Project Organizer. When you click on a file or folder, this bar shows an icon, as well as the name of the folder or file.
