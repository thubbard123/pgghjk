Conflict with Translation Memory Dialog

This dialog opens if you have already uploaded translated text to a translation memory (TM) database, later modify that text, and then attempt to upload the new translation to TM. Lingo recognizes this conflict and gives you different options for handling it. See About Translation Memory.

  1. Open an editor and modify the translation for text that has already been uploaded to your TM database. See Translating Files.
  2. Attempt to upload your changes to the TM database.

Button/Section

Description

Source Segment

Displays the text in the source language.

Your Translation

Displays the translation that you have provided. You can click in this field and modify the translation, if necessary.

Translation Memory Version

Displays the matching translation that is already uploaded into the TM database.

Change nothing

Select this option if you do not want to make any changes to the text you have modified or to the TM database. Your translation changes will be kept, and the old translation will continue to exist in the TM database.

Use your translation and overwrite translation memory

Select this option if you want to keep your modified translation and upload it to the TM database, replacing the old translation.

Use translation from TM

Select this option if you prefer to use the old translation that is already contained in the TM database. The text that you modified in the editor will revert back to the translation already contained in the TM database.

Apply to all conflicts

Select this check box if you want future conflicts such as this to be treated according to the option that you have selected.
