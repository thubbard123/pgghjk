Add File Dialog

This dialog is used to add the following files to a project. You can choose the kind of element from the File Type at the top of the dialog. When you do this, the dialog changes to display only the fields pertinent to adding that specific element.

  • TOC Read more… A table of contents (TOC) is a method that end users can use to find information easily in your output. A TOC in Flare works much like a TOC in a printed book. It provides an outline of contents in your Help system so that end users can easily and quickly navigate to the information they need. You create a TOC, adding books, topic links, and links to external files in a structure that you think would be useful for the user. End users then browse through a TOC to find information. For print-based () and output, you need to use the TOC Editor to create a TOC just as you would create one for online output (). However, there is a fundamental difference. Performing this task for online output creates an actual TOC in the output, which people use to navigate from topic to topic. This is not the case for print-based output. Performing this task for print-based output lets you indicate which topics will be included in the output and in what order. In that sense, this TOC functions more as an outline for print-based output. Therefore, for print-based output, you can think of it as an "outline TOC." If you want to include a generated TOC in print-based output, you need to use a TOC proxy in a topic instead. See Adding a Table of Contents.

Applying condition tags and File Tags to New Files

When you create new files, you can apply condition tags and file tags to them directly from the Add File dialog. This saves you the time of having to locate the file in the Content Explorer or Project Organizer after you create the file. To do this, expand the Attributes area at the bottom. Then next to the Condition Tags and File Tags fields, click and select the relevant conditions and file tags.

See Applying Conditions to Content and Associating Tags with Files.

 content files and the folder location

If you are adding a content file (e.g., topic, image, master page, multimedia, page layout, snippet, stylesheet, table stylesheet), the Folder field is automatically populated based on whatever has focus in the Content Explorer. Flare does not try to direct you to a particular folder (such as the traditional default folders for non-topic files). Instead, it assumes that you want to place the file in the folder that has focus in the Content Explorer, with the option to place it anywhere else if you like.

 tip

When adding a new file to the Content Explorer, the recommended method is to right-click on the folder in the Content Explorer and use the New menu option. This is the most efficient way to direct the new file to the folder where you want to store it.
