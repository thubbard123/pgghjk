Installing the Prerequisites for Windows Server 2012

To prepare for a Pulse installation, you need to prepare the systems that will be hosting the Pulse server application and the Microsoft SQL Server instance for the Pulse database.

Before You Begin

before you begin

In order to download and install the prerequisites, you need an Internet connection.

You also need the ability to log on the server(s) as a user who has access permissions to install software and change configuration settings on the system. This requires you to have access credentials for one of these accounts:

Installing the Prerequisites

installing the prerequisites

First, install Windows Server 2012 on the system acting as the Pulse web server and Pulse database server. Whether you are setting up a new server or just using an existing Windows server, you should also ensure that the operating system is updated with all of the latest service packs, critical patches, and security updates from Microsoft.

important

If you are not familiar with how to install a server operating system, contact your Windows Administrator and/or Network Administrator for assistance. This will help to ensure that your server is in compliance with any security and network requirements specific to your environment. See Pulse System Requirements for Windows Server 2012.

 How to Install IIS important note

IIS is a built-in component of the Windows OS. For Windows Server 2012, the OS lets you install IIS 8.0.

  1. Log in to the system that you selected to be the Pulse web server using one of these accounts:

  2. Determine if IIS is installed on the operating system. To do this, press the Windows key on your keyboard.

    Click the Server Manager tile to open the Server Manager.

    If IIS is installed, this should be indicated on the Server Manager.

  3.  If you need to install IIS on the server, add the Web Server (IIS) role service:how to add the web server (iis) role

    1. In the Server Manager, click Add roles and features.

      The Add Roles and Features Wizard opens.

    2. Click Next.

    3. Make sure Role-based or feature-based installation is selected. Then click Next.

      The local server should be listed.

    4. Click Next. The list of roles is shown.

    5. Scroll down and place a check mark in the Web Server (IIS) box.

      A dialog opens.

    6. Click Add Features.

      The Web Server (IIS) check box now has a check mark.

    7. Click Next.

      By default you should see options for .NET 3.5 and .NET 4.5. Only the option for .NET 4.5 should be selected.

    8. Scroll down and select Windows Identity Foundation 3.5.

    9. Click Next.
    10. Click Next again.

    11. Place a check mark next to all of the role services in the list below. When finished, click Next.
