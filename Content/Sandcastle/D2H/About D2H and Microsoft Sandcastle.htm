About Microsoft Sandcastle Integration with Doc-To-Help

Introduction

Microsoft's Sandcastle utility automatically creates MSDN style reference documentation from .NET assemblies and XML comment files. This makes it possible for you to document a class library—plus add narrative information—quickly and efficiently. Doc-To-Help integrates Sandcastle's XML output into your projects, automatically creating topics, index, TOC, and other elements. With this information added to your project, you can edit/add your own topics, and easily link to namespaces.

Doc-To-Help's integration with Sandcastle extends the functionality of the plug-in by providing an interface for Sandcastle, and giving authors more power to connect narrative and reference content.

To use the Doc-To-Help Sandcastle plug-in, you must first install Sandcastle and its patch. If you have an older version of Sandcastle on your machine, uninstall that version first.

How to Install Sandcastle

How to Install Sandcastle

  1. Download the Sandcastle Help File Builder version 2.7.10.

    Currently, Doc-To-Help only supports the official Microsoft release of Sandcastle.

  2. Run InstallResources\Sandcastle_2710.msi.

  3. After you have completed the Sandcastle Help File Builder installation, go to C:\Program Files (x86)\MadCap Software\DocToHelp\Plugins\Sandcastle.

  4. Double-click SandcastlePatch.exe to install it.

Tasks Associated With Sandcastle

Tasks Associated with Sandcastle

Following are the primary tasks associated with using Sandcastle in Doc-To-Help:

  • Create Project Using the New Project Wizard, you can create a new Sandcastle project. See Creating a Project with Sandcastle.
  • Convert Documenter for NET Projects If you have reference documentation originally created in Doc-To-Help’s Documenter for .NET, you can convert those projects to Sandcastle. See Converting Documenter for .NET Projects to Sandcastle Projects.
  • Configure Sandcastle Reference Section In the Properties for Generating Reference dialog, you can import a Documenter for .NET project file and/or configure your Sandcastle reference section. See Configuring the Sandcastle Reference Section.
  • Configure CodeBlock Component Dialog The Configure CodeBlock Component dialog is used to configure the elements of the CodeBlock component. See Configure CodeBlock Component Dialog.
  • Configure PostTransform Component Dialog The Configure PostTransform Component dialog is used to configure the elements of the PostTransform component. See Configure PostTransform Component Dialog.
  • CodeBlock and PostTransform Components The Doc-To-Help Sandcastle plug-in includes Eric Woodruff's components (with his permission) that complement Sandcastle functionality for better display of code blocks in various languages, including syntax highlighting and connecting code blocks to the language filter. Normally, you don't need to change the settings for those components, but the XML source document's Properties for Generating Reference allows you to do so (double-click on the plug-in document to open the Properties for Generating Reference dialog). See CodeBlock and PostTransform Components.
  • Exclude Elements without Description You can exclude elements that do not have descriptions (to add them to the filter). See Excluding Elements without Description.
  • Create Links from Narrative Text to Reference Text To create a link from your text to a generated topic (or to an added topic), all you need to do is to format a word or a phrase in your text (usually it is a name of a property, method, class, and so on) with an appropriate style. See Creating Links from Narrative Text to Reference Text.
  • XSLT Transformation of the Source Code Comments XML The XSLT file is specified in the Generation node of the Properties for Generating Reference dialog. To open the Properties for Generating Reference dialog, double-click on the plugin document.
 Note

Class libraries cannot be exported to Manual or EPUB Targets. It is suggested you exclude the Sandcastle plug-in from those outputs using a condition. See Setting Document Properties. See the online Help.
