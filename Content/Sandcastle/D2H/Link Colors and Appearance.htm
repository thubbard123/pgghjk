Link Colors and Appearance

By default, links use the color blue and underline. You can change this, as with many other appearance and formatting style attributes, by modifying the target template used by Doc-To-Help. The Sandcastle styles are unique to the D2H_sandcastle_src.dotx template file, so in this case you can also modify the style in the source template. See About Templates and Styles. See the online Help or the Making It Look Good Guide.

Standard Link Colors

Blue underlined links are an MSDN standard, so it was chosen as the default in Sandcastle. But you can change the colors, or you can make links look like standard links in the corresponding target. To use your custom link appearance, verify the Affects appearance check box in the Project Styles dialog in selected for the desired character style. To use the standard link appearance instead of the one specified by styles in target templates, verify that the Affects appearance check box in the Project Styles dialog is cleared. See About Project Styles and Topic Types.

For targets that are based on HTML and viewed with Internet Explorer (or standalone Microsoft viewers, such as Help 2.0 or HTML Help), standard link colors are determined by the Internet Explorer settings (Internet Options>General>Colors).
