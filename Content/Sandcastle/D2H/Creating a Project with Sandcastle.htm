Creating a Project with Sandcastle

Using the New Project Wizard, you can create a new Sandcastle project.

How to Create a New Sandcastle Project
  1. Create a new Doc-To-Help project using the New Project Wizard (File>New Project).
  2. Select Project>Project Styles. The Project Styles dialog opens.
  3. For paragraph styles Heading 1-Heading 5 on the left, change the Topic type to Conceptual.
  4. Select Project>Plugins and choose Create New Plugin Document to add a new Sandcastle plug-in document.
  5. For the .xmlprop file in the XMLDocuments subdirectory, enter a name, such as Reference.
  6. From the Choose Plugin Type dialog, select the Sandcastle plugin type and click OK. A new document (XMLDocuments\Reference) is created.
  7. Double-click Reference. The Properties for Generating Reference dialog opens.
  8. Select the assemblies you want to document and the source XML comment file(s) for them. You can also specify the framework version, the types and members to include, and the syntax to include.

  1. In the Documents window pane, right-click on Reference and choose Generate. This generates the Reference.xml file.

    If any of the dependent properties are changed later, Doc-To-Help will automatically regenerate the reference section when the  plug-in is closed, or when building the target.

  2. Open the file your narrative will reside in.

    If it is not using the D2H_sandcastle_src.dotx template, you should apply it now. See Creating Links from Narrative Text to Reference Text.

  3. Add text.
  4. If you would like to create a link from the narrative document to the reference document, highlight the text to link and select the appropriate toolbar button or style.
  5. Select Home>Select Target to choose the target.
  6. Click the Rebuild Target button. The target will be formatted with the MSDN reference style.
