Configuring the Search Database

In a Flare target, you have a variety of options that you can enable or disable to define how the search database performs.

  • Chunk Large Search Database Files You can select this option to define the maximum size for database chunk files. See Chunking Large Search Database Files.
  • Index Entries—Excluding From Search If you insert index keyword markers in your topics, Flare automatically takes those keywords into account when retrieving and ranking search results. You can exclude index markers from search. See Excluding Index Entries from Search.
  • N-Gram Size N-grams can be applied to the search database to improve its ability to retrieve more accurate matches for Chinese, Japanese, and Korean languages. See Setting the N-Gram Size for Search.
  • Non-Words—Excluding From Search Flare includes a hard-coded list of non-words that can be omitted from search (e.g., angle brackets [ ] ). See Excluding Non-words from Search.
  • Partial-word Searching You can enable a partial-word search capabilities. It is important to note that enabling this option can greatly increase the size of the output. See Enabling Partial-word Search.
  • Pre-merge Search Database File If you have multiple projects and plan to merge their outputs when building the "master" project, you can pre-merge the search database files. See Pre-merging Search Database Files and Merging Projects.
  • SQL Search Database— If you have installed Microsoft SQL Server Compact on the system building the target output, you have the option to include an SQL database to support search with the published output. This can improve search performance and permit users to use the asterisk (*) wildcard with search. See Including an SQL Search Database.
  • Stop Words—Excluding in Search Flare includes a hard-coded list of English stop words that can be used to omit certain words (e.g., "an," "and," "or," and "the") from the search database. See Including Stop Words in Search.
