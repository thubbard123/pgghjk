 Conditions—Add File Dialog Content Explorer and Project Organizer—Filter and Multi-Select

When working in the Content Explorer and Project Organizer, you will notice some enhancements. These have to do with a new and improved filter, as well as the ability to select multiple files.

Filter

The old filter in the local toolbar of the Content Explorer has been replaced. The Project Organizer did not previously have a filter, but now it uses one similar to that in the Content Explorer.

Under the toolbar buttons, you will notice a plain text field. As you type text in this field, the folders and files below are reduced, showing only those that match what you’ve entered in the field. This makes it much easier to find files. To see all of the folders and files again, simply clear the filter field.

Example

Let’s say your Content Explorer looks like this:

You need to find a file that contains the word “link” in the file name. You’re not sure where you placed the file, and you’re not even sure of the full name for the file. You only know that it contains the word “link.”

So in the filter field, you type link. As a result, the folders and files are limited to only those containing that word, which is highlighted.

 Multi-Select

Previously, you could not select multiple folders or files in the Content Explorer or Project Organizer unless you used the split view feature. See Content Explorer and Project Organizer.

Starting with this version, you can hold down your SHIFT or CTRL key and select multiple folders and files in the regular view of the Content Explorer or Project Organizer.

By selecting multiple folders or files, you can now accomplish more from the Content Explorer or Project Organizer, including the following:

  • Open multiple files at once
  • Drag multiple files to another location or to an editor
  • Apply property settings (such as conditions) to multiple folders or files at once
 Note

Although multiple files can now be selected either in the regular view or the split view, the split view option is still available in Flare. The primary reason for this is that the split view feature also lets you see many details for files (e.g., size, date modified, title) using columns. See Viewing File Details.

File Tags—Add File Dialog Google Search

As an alternative to MadCap Search, you can now use Google Search for your HTML5 Top Navigation output. There are benefits of both the Google and MadCap Search methods.

 

Google Search

MadCap Search

Quality

Results are typically better, depending on the search performed.

Results might not be as robust, depending on the search performed.

Public Versus Local Output

Output must be available publicly so that you can use Google. Output cannot be behind a firewall.

Output can be searched whether it is available publicly or locally.

General Speed

Search results might lag at times, depending on the size of the output and whether Google is finished indexing all of it.

(There may be ways to prod Google to index via their UI or using our sitemap feature.) Ask Rickxxx

Search results might be faster because the output is immediately indexed once it is generated.

Mobile Devices

Search results always come from a server, so you do not need to be concerned about the ability of the mobile device to handle the information.

Search results are "chunked" and then collapsed so that they can be displayed when a search is performed. This is usually okay on PCs, but if you try to perform a search on a mobile device, the results might be slow or even crash because the device cannot handle the number of search results.

Using Google Search requires a combination of steps on the Internet, as well as in Flare:

Internet Steps

  1. Go to cse.google.com and log in with your Google (i.e., gmail) password.

  2. Create a custom search engine for the site where your output will be published.

  3. When you're done, copy the key that you see at the end of the URL.

Flare Steps

  1. Open an HTML5 target.
  2. Select the Search tab.
  3. From the Search Engine field, select Google Search.

  4. In the Google Search Context Key field, paste the key that you copied from cse.google.com.

Google Search is not supported for HTML5 Tripane output.

The Search Engine Optimization fields have been moved in the Target Editor from the Advanced tab to the new Search tab. This includes fields for the sitemap, importance, glossary, results per page, and abstract.

Thesaurus Window Pane and Synonyms Context Menu

A Thesaurus window pane has been added to Flare, allowing you to quickly view and select words related to those in an open topic or snippet. See xxx.

You can open the Thesaurus window pane by opening the Tools ribbon and choosing Thesaurus. Following a search field—which lets you manually look for a term—you will see categories of related terms in the Thesaurus window pane. You can expand or collapse any of these categories by clicking the arrow buttons. If you click on one of the terms in a category, the window pane adjusts, showing different words and categories specific to that term.

To use a term provided in the Thesaurus window pane, you can right-click the term and select Insert in the context menu. This replaces the word nearest the cursor in the open topic or snippet. You can also select Copy, then paste the term anywhere you like.

You can also right-click a term in the open topic or snippet, then click Synonyms, and choose a word in the context menu.

The information that displays in the Thesaurus window pane and the Synonyms context menu comes from the dictionary you are using in your project (see About Dictionaries). Keep in mind that only some of the available dictionaries also provide a thesaurus. If you are using a dictionary without a thesaurus, the Thesaurus window pane will be empty when you open it.
