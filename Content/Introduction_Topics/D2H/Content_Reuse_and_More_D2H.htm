Content Reuse and More

There are all kinds of things you can do in . But you don't have to do them all. There are actually just a few concepts and features that are the most important to grasp. Don’t expect to be an expert at everything right away. Instead, begin by trying to wrap your brain around a few concepts and features that are most important when it comes to learning . These concepts and features all tend to revolve around the idea of content reuse (or single-sourcing), which means that you can take the same content, re-use it, and produce multiple outputs from it.

Examples

Examples

One of the most common ways to single-source a project is to produce one type of online output (such as NetHelp), as well as multiple print-based outputs (Manual targets) from the same source Word document.

Another way to use single-sourcing is to use one set of documents to create documentation for multiple audiences, such as an Administrator version and a Manager version.

One more possibility is that you might take the same source content and generate output for different purposes, such as documentation, training, or reference materials.

You might even use a combination of these techniques. For example, you might create one desktop-based online Help system, a web-based elearning site, a PDF quick start manual for administrators, a PDF quick start manual for managers, and a PDF reference manual for co-workers. And it is all created from the same Word source content.

Following are some important questions you will likely need to answer before you begin creating files and content:

  • What kinds of output do you want to create? Do you need online output, print output, or both? How many different versions of online and print output do you need? See Output Types and Targets. See Developing Targets.
  • How will you name your files to keep things consistent and organized?
  • What kinds of variables will you need? See Variables.
  • What kinds of conditions will you need? See Conditions.
  • Will one author be working on the project or will multiple people be involved? See About Team Authoring. See the online Help or the Managing Your Project Guide.

In other words, the more planning and preparation you do beforehand, the smoother your journey in will be.

 Variables

Variables allow you to write content once, and manage it in one place for reuse across your project. You might create text variables for often-used text that does not need to be formatted (e.g., product or company name, address, version number, system date/time). Alternatively, you can use rich content variables for blocks of formatted content (e.g., tables, images, entire topics). See About Variables.

 Conditions

This is where gets really powerful. Conditions are used to tell , "Hey, this piece of content or this file—I want it to be in this output but not that one." For example, maybe you plan to have two versions of some online output—one version for beginners and one version for advanced users. Much of the content you create is going to be the same for both audiences, but some of it is going to be written only for a specific audience. So you can create conditions for each audience and use them to separate the content when you build your output. See About Conditions.

 Templates

Doc-To-Help uses two kinds of predefined templates (DOTX files) to determine how content will look.

  • Source Templates These control the look of source documents (when you are authoring in Word). Additionally, all of the content in source templates appears in new documents created from the template, so you can use customized source templates as a way to provide structure to new documents (e.g., defining headings, providing sample text).
  • Target Templates These control the look of the final output.

These templates contain styles that you apply to your Word source document. The source styles are interpreted by the target template you have chosen so that your output looks and behaves the way you desire. Styles control your final outputs in multiple ways, and they are a convenient and efficient way to keep your content looking consistent. Therefore, it is important to use them and avoid applying local formatting.

For more information, see About Templates and Styles.

For more information, see the online Help, the Making It Look Good Guide, or the Documents, Templates, and Content Guide.
