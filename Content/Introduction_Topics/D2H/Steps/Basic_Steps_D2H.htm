Basic Steps for Creating and Developing a Project

When you break down the authoring process in , you will discover that it can be quite simple. Following are the basic steps that you need to follow for creating and developing a project in .

  1. Create or Convert a Project Create a new project, or convert a legacy project from RoboHelp Word, WinHelp, or 2000. See Creating and Converting Projects.
  2. Work With Documents, Templates, and Content Add Word source documents based on special Doc-To-Help templates. Then add content and objects, such as text, images, tables, navigation links, and more. You can also add other kinds of elements—such as glossaries, indexes, conditions—to your project. See Work With Documents, Templates, and Content.

  3. Make It Look Good There are multiple ways to affect the way your output looks. This includes styles and templates (for formatting content), as well as themes and window display (for controlling the area around the content in online output—the "skin"). See Making It Look Good.

  4. Develop Targets Decide the type(s) of output formats that you want to generate, design targets, and set properties accordingly to meet your needs. See Developing Targets.
  5. Build Output Select a target and generate the file(s) that you will deliver to users. See Building Output.
